## 1. 模块概述

电位器管线负责 4 路 100mm 推杆的完整信号处理链：从 ADC 硬件采样到 MIDI 事件生成。管线内部严格遵循 HAL → CORE → APP 三层结构。

**关键设计决策**：  
《MIDI 控制器自描述配置协议 v2.6》中的 **14-bit 校准值编码**是纯粹的 **SysEx 传输层格式**。接收端解码后，对于 12-bit ADC，其有效值域仍为 **0–4095**。因此本管线内部所有算法均在 **12-bit 运算域** 进行。

## 3. 硬件抽象层设计（HAL）

### 3.1 职责
- 管理 4 路 ADC 引脚（GPIO26–29）及 RP2040 ADC 外设。
- 实现算法要求的“切换通道后连续采样 N+2 次，后 N 次算术平均”的采样策略。
- 为上层提供单通道 **12-bit 原始采样值（0–4095）**。

### 3.2 宏配置
```c
#define POT_HAL_SAMPLE_COUNT  16   // N，默认 16。**设计约束：必须为偶数**，以保证四舍五入精度。
#define POT_HAL_DISCARD       2    // 丢弃的前 2 次采样
```

### 3.3 内部接口
```c
void pot_hal_init(void);
uint16_t pot_hal_read_channel(uint8_t channel); // 返回 12-bit 值 (0–4095)
```

### 3.4 初始化序列
`pot_hal_init()` 必须严格按以下顺序执行：
1. `adc_init();`                    // 初始化 ADC 时钟（48 MHz）
2. `adc_gpio_init(26);`             // 初始化 GPIO26，禁用数字输入缓冲，设为模拟模式
3. `adc_gpio_init(27);`
4. `adc_gpio_init(28);`
5. `adc_gpio_init(29);`

遗漏 `adc_gpio_init()` 会导致数字输入缓冲漏电流，在 10 kΩ 电位器上产生 10–50 mV 偏移（约 12–60 LSB 误差），严重影响校准精度。

### 3.5 采样算法实现
```c
uint16_t pot_hal_read_channel(uint8_t channel) {
    // channel 0-3 直接对应 ADC 输入 0-3 (GPIO26-29)
    adc_select_input(channel);

    uint32_t sum = 0;
    for (int i = 0; i < POT_HAL_SAMPLE_COUNT + POT_HAL_DISCARD; i++) {
        uint16_t val = adc_read();   // 12-bit
        if (i >= POT_HAL_DISCARD) {
            sum += val;
        }
    }
    // 四舍五入平均，依赖 POT_HAL_SAMPLE_COUNT 为偶数以保证精度
    return (uint16_t)((sum + POT_HAL_SAMPLE_COUNT / 2) / POT_HAL_SAMPLE_COUNT);
}
```

## 4. 核心算法层设计（CORE）

### 4.1 职责
CORE 层实现与硬件无关的迟滞比较与变化判定逻辑，所有数值均在 **12-bit 运算域**。

### 4.2 状态数据结构
```c
typedef struct {
    uint16_t last_raw;       // 上次有效发送时的原始值 (12-bit 域)
    uint8_t  last_midi;      // 上次发送的 MIDI 值 (0–127)
} pot_core_state_t;
```
- `last_raw` 初始值 `0xFFFF`：哨兵值（大于 12-bit 最大值 4095），表示“从未发送”，确保首次任意有效输入都能触发事件。
- `last_midi` 初始值 `0xFF`：哨兵值（超出 MIDI 0–127 有效域），确保首次任意 MIDI 值均能触发事件。**禁止**将其初始化为 0–127 范围内的任何值。

### 4.3 宏配置
```c
#define POT_CORE_HYSTERESIS  16   // 迟滞区 M（12-bit LSB），对应 100mm 行程约 0.2mm
```

### 4.4 内部接口
```c
void pot_core_init(pot_core_state_t *state);

/**
 * @brief 对原始值进行迟滞比较与 MIDI 映射
 * @param raw_in_12bit    当前滤波后的原始值。**前置条件：必须为有效 ADC 读数（0–4095），调用者负责过滤错误值（如 0xFFFF）。**
 * @param cal_min_12bit   校准最小值（12-bit 域）
 * @param cal_max_12bit   校准最大值（12-bit 域）
 * @param state           该通道的状态记录（会更新）
 * @param midi_out        输出参数，仅当返回 true 时有效
 * @return                是否需要发送新 MIDI 事件
 */
bool pot_core_process(uint16_t raw_in_12bit,
                      uint16_t cal_min_12bit, uint16_t cal_max_12bit,
                      pot_core_state_t *state,
                      uint8_t *midi_out);
```

### 4.5 初始化与重置
`pot_core_init()` 内部直接调用 `pot_core_reset()`，将 `last_raw` 设为 `0xFFFF`，`last_midi` 设为 `0xFF`。  
`pot_core_reset()` 提供同样的重置能力，供配置变更后调用。

### 4.6 迟滞与映射算法
```c
bool pot_core_process(uint16_t raw_in_12bit,
                      uint16_t cal_min_12bit, uint16_t cal_max_12bit,
                      pot_core_state_t *state,
                      uint8_t *midi_out)
{
    // 0. 防御：校准值无效时直接放弃（fail-fast）
    if (cal_max_12bit <= cal_min_12bit) {
        return false;
    }

    // 1. 迟滞判断
    uint16_t last = state->last_raw;
    if (last != 0xFFFF) {  // 非首次，检查死区
        int32_t diff = (int32_t)raw_in_12bit - (int32_t)last;
        if (diff >= -(POT_CORE_HYSTERESIS/2) && diff <= (POT_CORE_HYSTERESIS/2)) {
            return false;  // 未超出死区
        }
    }

    // 2. 线性映射到 MIDI (0-127)
    uint8_t midi;
    if (raw_in_12bit <= cal_min_12bit) {
        midi = 0;
    } else if (raw_in_12bit >= cal_max_12bit) {
        midi = 127;
    } else {
        uint32_t range = cal_max_12bit - cal_min_12bit;
        midi = (uint8_t)(((uint32_t)(raw_in_12bit - cal_min_12bit) * 127 + range/2) / range);
    }

    // 3. MIDI 变化检测
    if (midi == state->last_midi) {
        // *** 关键设计约束 ***
        // 此处绝不更新 last_raw。
        // 若更新，死区将变为“滑动窗口”，慢速移动时 last_raw 始终追踪 raw 值，
        // 累积偏移永远无法跨越 MIDI 量化步长，导致推杆全程仅输出首尾两个 MIDI 值。
        return false;
    }

    // 4. 更新状态并输出
    // last_raw 仅在 MIDI 值真正变化时才更新，保证死区以“上次有效发送位置”为中心。
    state->last_raw = raw_in_12bit;
    state->last_midi = midi;
    *midi_out = midi;
    return true;
}
```

## 5. 应用层设计（APP）

### 5.1 职责
- 组合 HAL 和 CORE，完成 4 通道全流程处理。
- 通过市场 API 获取各通道的校准值和 MIDI 映射（`config_get_calibration` 返回的 `cal_min/cal_max` 已是解码后的 12-bit 值）。
- 管理暂停/恢复和事件缓存。
- 实现 `pot_api.h` 中所有接口。

### 5.2 内部数据结构
```c
static struct {
    bool paused;
    bool event_pending[POT_COUNT];
    uint8_t event_channel[POT_COUNT];
    uint8_t event_cc[POT_COUNT];
    uint8_t event_value[POT_COUNT];
    pot_core_state_t core_states[POT_COUNT];
} pot_ctx;
```

### 5.3 对外 API 实现

#### `pot_init()`
- 调用 `pot_hal_init()`。
- 遍历 4 通道，调用 `pot_core_init(&pot_ctx.core_states[i])`。
- `paused = false`，清除所有事件标志。

#### `pot_poll()`
```c
void pot_poll(void) {
    if (pot_ctx.paused) return;

    // 1. 拉取配置快照（批量接口保证一致性）
    pot_mapping_t mappings[POT_COUNT];
    pot_calibration_t cals[POT_COUNT];
    uint8_t count;

    // 校验返回值，失败则本轮不进行采样，也不产生任何事件。
    // 若配置管线持续异常，本函数将永久静默，设备表现为“推杆无响应”。
    // 此风险需由系统级健康检测（如 LED 状态机或上位机查询）覆盖。
    if (!config_get_all_pot_mappings(mappings, &count) || count < POT_COUNT) return;
    if (!config_get_all_calibrations(cals, &count) || count < POT_COUNT) return;

    // 2. HAL 采样（12-bit）
    uint16_t raw_12[POT_COUNT];
    for (int i = 0; i < POT_COUNT; i++) {
        raw_12[i] = pot_hal_read_channel(i);
    }

    // 3. CORE 处理 + 事件缓存
    for (int i = 0; i < POT_COUNT; i++) {
        uint8_t midi_val;
        if (pot_core_process(raw_12[i], cals[i].cal_min, cals[i].cal_max,
                             &pot_ctx.core_states[i], &midi_val)) {
            pot_ctx.event_pending[i] = true;
            pot_ctx.event_channel[i] = mappings[i].channel;
            pot_ctx.event_cc[i] = mappings[i].cc;
            pot_ctx.event_value[i] = midi_val;
        }
    }
}
```
**性能注释**：每 10 ms 查询一次配置，开销极小，可保证配置热更新立即生效。

#### `pot_get_midi_event(uint8_t index, uint8_t *channel, uint8_t *cc, uint8_t *value)`
- **前置条件**：必须在 `pot_poll()` 之后、下一次 `pot_poll()` 之前调用。违反此约定将导致行为未定义（重复发送或丢失事件）。
- 检查 `pot_ctx.event_pending[index]`。
- 若为 true，拷贝缓存数据，清除标志，返回 true；否则返回 false。

#### `pot_get_raw(uint8_t index)` / `pot_get_all_raw(...)`
- 若管线暂停，返回 `0xFFFF`（错误指示）。
- **调用者陷阱**：`0xFFFF` 不是有效 ADC 值。若调用者未校验此错误值并传入 CORE 层，将导致非法映射（输出 MIDI 127）。调用者（如 `cmd_cfg_app` 处理 0x11 命令）必须校验返回值有效性。
- **互斥安全性说明**：`pot_hal_read_channel` 每次调用均执行完整的 `adc_select_input(channel)` → 采样 → 平均流程。即使与 `pot_poll()` 交叉调用，由于每次采样前都会重新选择正确通道，不会发生通道错位。安全基础在于 HAL 层的通道选择幂等性。

#### `pot_set_pause(bool pause)`
- 设置 `pot_ctx.paused`，无额外动作。

#### `pot_reset_stable_values(void)`
- 遍历所有通道调用 `pot_core_reset()`。
- 清除所有 `event_pending` 标志，避免残留事件。

## 6. 并发与数据一致性

- 单核主循环模型，所有全局状态修改均在 `pot_poll` 内同步发生，无并发竞态。
- 配置快照使用双缓冲+指针原子切换，`config_get_all_*` 查询保证单次调用内数据一致性。
- 暂停标志 `volatile bool` 在 `pot_poll` 入口检查，修改仅由主循环内 `cmd_cfg_task` 触发，无竞争。

---

## 7. 性能指标

- 单路 HAL 采样时间：~36 μs（18 次 ADC 转换）
- 4 路全流程（采样+映射+迟滞）：< 200 μs
- 端到端延迟（物理推杆移动 → MIDI CC 入队）：< 10 ms

## 8. 可测试性

- CORE 层完全无 HAL 依赖，可独立进行单元测试，验证迟滞边界、线性映射精度、反复震荡抑制。
- HAL 层可通过模拟 ADC 读取函数进行测试。
- APP 层测试时可注入模拟的 CORE 和配置 API，验证事件缓存、暂停/恢复逻辑。
