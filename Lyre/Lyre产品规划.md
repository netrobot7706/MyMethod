# Lyre 产品规划

## 定义
- Lyre项目是一个四个推杆的MIDI控制器

## 硬件（已经完成打版与焊接，整体组装完成）
- Lyre 使用RP2040-Zero开发板作为控制核心。
- 开发板的3V3电源经过22R电阻+10uf+0.1uf之后净化为V_clear，用于给电位器供电。
- 四个10CM行程的电位器，两头分别接V_clear与GND，四个抽头分别经过1KR+0.1uf的RC滤波器之后接入RP2040的A1~A4（GPIO26~29）。

## 需要实现的软件功能

### 固件端软件
- RP2040需要枚举成USB MIDI + USB CDC的复合设备。通过MIDI口给电脑发送MIDI消息，通过USB CDC与上位机配置器通信。
- USB CDC通信的协议已有，作为附件。
- 由于RP2040的ADC具备足够的采样率，同时采样精度有限。固件需要实现过采样、迟滞等功能，目标是在不动推杆的时候，控制器不能发送幽灵消息。
- MIDI消息采用差分发送，只有在与上次发的消息不一致时才发送，防止消息风暴。（Control Surface库可能已经有实现）
- 固件使用Arduino IDE + TinyUSB开发。
- 固件能够接收上位机发送过来的配置，按配置在配置中设置的channel上发送MIDI CC#。
- 配置需要在本地持久化，下次开机时不用重新设置。注意保护Flash，减少不必要的写入。
- 为了简化固件开发，尽量使用现有的库，如Control Surface。

### 上位机端软件
- 使用Web前端开发。推荐Vue+Typescript+Tailwind。
- 通过Web Serial与固件通信。握手：连接后需要获取设备信息与版本。见通信协议。
- 设置页面上，需要有设备选择框，连接按钮。连接上了之后，要有一个标志标示，比如把什么控件的颜色变成绿色之类。这时候连接按钮变成断开按钮。连接成功后，上位机应主动向固件请求当前保存在设备里的配置（CC#和Channel#），并刷新到网页UI上，使用户看到当前设备的实际状态。
- 设置页面上，有四个方框，每个框内是电位器的设置项。设置项有cc#，Channel#，以及一个临时变化按钮。这个按钮只是临时变化，固件不作持久化。点击后立即通过CDC发送给固件并生效，允许用户立刻在DAW（音频工作站）里测试效果。但如果不点‘保存到设备’，设备断电重启后将恢复为上一次保存的值。
- CC# 设置除了可以输入数字之外，还提供下拉菜单供用户选择标准MIDI CC，全语义配置，免去用户记忆查表操作。
- 设置页面最下方，是恢复出厂设置，与”保存到设备”按钮。点击保存到设备之后，一次性把所有配置传输到设备并持久化。
- 出厂默认设置，四个通道的MIDI cc分别是1，2，11，13。均为通道1

# Lyre 产品规划 (用于生成PRD)

## 1. 项目定义
- Lyre项目是一个带有四个10CM行程推杆的MIDI控制器。

## 2. 硬件架构
- 控制核心：RP2040-Zero开发板。
- 电源净化：开发板3V3电源经22R电阻 + 10uf + 0.1uf滤波为V_clear，专用于电位器供电。
- 信号输入：4个10CM电位器，两端接V_clear与GND，抽头经1KR + 0.1uf的RC滤波器接入RP2040的A1~A4（GPIO26~29）。

## 3. 固件端功能需求 (Firmware)
- **设备枚举**：基于Arduino IDE + TinyUSB，枚举为 USB MIDI + USB CDC 复合设备。
- **数据防抖与滤波**：针对RP2040 ADC特性，需在固件层实现过采样（Oversampling）与迟滞（Hysteresis）算法。目标：静止状态下绝对不触发“幽灵消息”；推杆物理两端能完美映射到MIDI的 0 和 127。
- **MIDI发送逻辑**：仅在推杆数值发生有效改变时，在配置的Channel上发送对应的MIDI CC#消息。防止发送同一数值MIDI CC，造成消息风暴。
- **配置通信**：通过USB CDC接收上位机指令（协议格式待设计）。支持：读取配置、临时测试配置、保存配置、恢复出厂设置。
- **本地持久化**：使用Flash模拟EEPROM（如LittleFS）存储配置。默认出厂配置为：推杆1~4对应CC# 1~4，Channel统一为1。
- **技术栈**：Arduino IDE, TinyUSB, Control Surface库。

## 4. 上位机端功能需求 (Web App)
- **技术栈**：Vue 3 + TypeScript + Tailwind CSS。
- **通信机制**：通过 Web Serial API 与固件通信。
- **连接管理**：
  - 包含设备选择框、连接/断开按钮。
  - 连接成功后需进行握手逻辑（上位机发查询，固件回传设备ID），验证通过后状态指示灯变绿，按钮变为“断开”。
  - 连接成功时，自动从固件读取当前配置并刷新到网页UI。
- **配置面板**：
  - 包含4个独立方框，对应4个推杆。
  - 每个方框可设置：MIDI CC# (0-127)、Channel# (1-16)。
  - 每个方框有一个“临时测试”按钮：点击后将配置发送至固件立即生效，但固件不写入Flash，断电失效。
- **全局操作**：
  - 页面底部设“保存到设备”按钮：点击后将4个推杆的配置一次性打包发送给固件，固件执行持久化写入。
  - 页面底部设“恢复出厂设置”按钮：点击后将设备重置为默认值并刷新UI。
- **异常处理**：
  - 浏览器不支持Web Serial时显示兼容性提示。
  - 通信超时或意外断开时，提示用户并恢复未连接UI状态。
- **校准功能**：
  - 配置器提供校准功能，可以将每个电位器的最大值与最小值持久化到固件。
