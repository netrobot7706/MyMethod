# MIDI 控制器自描述配置协议 v2.6

## 基于库与虚拟控件的统一配置模型

---

## 1. 概述

本协议定义了一套基于 **MIDI 系统专用信息 (SysEx)** 的控制器自描述与配置机制。它采用 **库 (Bank) 与虚拟控件 (Virtual Control)** 的统一模型，能够同时适配从最简单的固定功能旋钮盒到带库切换的复杂推子控制器的所有产品形态。

v2.6 完善了上位机辅助校准流程，并修正了示例和长度计算中的若干错误，文档已自包含，无需参考旧版本。

---

## 2. 核心概念

| 概念 | 说明 |
|------|------|
| **物理控件** | 实际焊接在 PCB 上的旋钮、推杆、按钮。通过 `0x04` 和 `0x08` 描述其电气特性、校准值和面板布局。物理映射由设备硬件决定，固件启动时加载，**不支持上位机修改**。 |
| **库 (Bank)** | 一组虚拟控件的集合。设备可有 1~127 个库。 |
| **虚拟控件** | 在特定库中绑定到一个物理控件，并具有独立 MIDI 配置（CC 号、通道）的逻辑单元。 |
| **单库设备** | 库数量 = 1，所有虚拟控件与物理控件一一对应。 |
| **多库设备** | 库数量 > 1，一个物理控件可在不同库中映射为不同的 MIDI 控制。 |
| **校准数据** | 每个物理控件（旋钮、推杆）的 ADC 原始值最小/最大边界，用于将原始 ADC 读数线性映射到 MIDI 0–127。采用 **14-bit 编码**，范围 0–16383。按钮控件不参与线性校准。 |

**统一关系：**
```
物理控件 ←1对多→ 虚拟控件 ←多对1→ 库
```

**按钮的行为模型：**
按钮在布局树中以 `0x12` 类型出现。其行为由是否出现在虚拟控件表中决定：
- 若**不在**虚拟控件表中，行为由固件定义（如切库）。
- 若**在**虚拟控件表中，行为遵循该表的 CC/通道配置。
- 一个按钮不应同时承担两种角色。

## 3. SysEx 协议基础

### 3.1 消息结构
```
F0 7D <设备ID> <命令字> [有效载荷] <校验和> F7
```
- **厂商 ID**：`0x7D`
- **设备 ID**：`0x00` – `0x7E`（严禁使用 `0x7F`）
- **数据字节约束**：所有字节 bit7 必须为 0。

### 3.2 校验和
Roland 式求和补码：
```
sum = (待校验字节之和) & 0x7F
checksum = (128 - sum) & 0x7F
```
校验范围：从 `7D` 到校验和前一字节。

### 3.3 消息长度
- **虚拟控件命令**（0x0B–0x0E）：V ≤ 126，最大消息 8 + 4V ≈ 512 字节。
- **物理信息命令**（0x03/0x04）：N ≤ 127，每控件 6 字节，消息长 = 8 + 6N，最大 770 字节。
- **校准写入命令**（0x0F）：N ≤ 127，消息长 = 7 + 4N，最大 515 字节。
- **ADC 原始值查询**（0x11/0x12）：0x12 响应长 = 7 + 2N，最大 261 字节。

建议实现者按最大 N=127 分配缓冲区，确保容纳所有消息。

## 4. 命令字定义

| 命令字 | 方向 | 功能 |
|--------|------|------|
| `0x03` | PC → 设备 | 查询物理设备信息 |
| `0x04` | 设备 → PC | 返回物理设备信息（数量、映射、校准） |
| `0x07` | PC → 设备 | 查询面板布局描述 |
| `0x08` | 设备 → PC | 返回面板布局描述 |
| `0x0B` | PC → 设备 | 查询虚拟控件与库配置 |
| `0x0C` | 设备 → PC | 返回虚拟控件与库配置 |
| `0x0D` | PC → 设备 | 设置虚拟控件与库配置（写入） |
| `0x0E` | 设备 → PC | 虚拟控件写入应答 |
| `0x0F` | PC → 设备 | 设置物理控件校准数据 |
| `0x10` | 设备 → PC | 校准写入应答 |
| `0x11` | PC → 设备 | 查询物理控件 ADC 原始值 |
| `0x12` | 设备 → PC | 返回物理控件 ADC 原始值 |

> **命令字与布局节点类型码冲突说明**：`0x10`（旋钮类型码 / 校准应答）、`0x11`（推杆类型码 / 查询原始ADC）、`0x12`（按钮类型码 / 返回原始ADC）数值重叠，但它们位于 SysEx 消息的不同字段，解析器不会混淆。

## 5. 物理设备信息（命令 `0x03` / `0x04`）

### 5.1 查询
```
F0 7D <ID> 03 <校验和> F7
```

### 5.2 响应
```
F0 7D <ID> 04 <物理控件数量 N> <协议版本> <物理描述数据块> <校验和> F7
```
- **N** (1 字节)：1–127
- **协议版本** (1 字节)：当前为 `0x16`（v2.6）
- **物理描述数据块**：每控件 **6 字节**，共 6N 字节，按物理索引 0～N-1 排列。

每控件 6 字节结构：

| 偏移 | 字段 | 说明 |
|------|------|------|
| +0 | 多路器索引 | `0x00` = 直连 ADC，`0x01`–`0x7F` = 多路器芯片编号 |
| +1 | 通道号 | 直连时 ADC 通道 0–15；多路器时芯片输入通道号 |
| +2 | cal_min_mid | 校准最小值 高 7 位（14-bit 编码） |
| +3 | cal_min_lo | 校准最小值 低 7 位 |
| +4 | cal_max_mid | 校准最大值 高 7 位 |
| +5 | cal_max_lo | 校准最大值 低 7 位 |

**14-bit 校准值编码：**
```
value（0–16383） → mid = (value >> 7) & 0x7F, lo = value & 0x7F
解码：value = (mid << 7) | lo
```

**出厂默认值：** cal_min = `0x00 0x00`（0），cal_max = `0x1F 0x7F`（4095）。当 cal_min==0 且 cal_max==4095 时，视为未校准，可跳过线性映射以保证即插即用。

**示例（8 推子，ID=0，无多路器）：**
```
F0 7D 00 04 08 16
   00 03  00 00  1F 70   // 物理0: ADC3, min=0, max=4080
   00 01  00 19  1F 7F   // 物理1: ADC1, min=25, max=4095
   ... (其余6个)
   <校验和> F7
```

> **注**：物理映射（多路器索引、通道）由固件固化，不支持上位机修改。校准数据可通过 `0x0F` 命令更新。按钮控件的校准值无实际意义，但 `0x04` 仍会返回其默认值。

## 6. 面板布局描述（命令 `0x07` / `0x08`）

### 6.1 查询
```
F0 7D <ID> 07 <校验和> F7
```

### 6.2 响应
```
F0 7D <ID> 08 <长度高> <长度低> <树字节流> <校验和> F7
```
- 长度字段：2 字节，14-bit MIDI 编码，表示树字节流的字节数。

### 6.3 节点类型编码

| 类型码 | 含义 | 后续数据 |
|--------|------|----------|
| `0x01` | 水平容器 (HBox) | 1 字节子节点数 n，后跟 n 个子节点 |
| `0x02` | 垂直容器 (VBox) | 同上 |
| `0x03` | 网格容器 (Grid) | 1 字节列数，1 字节行数，后跟 cols×rows 个子节点（行优先），乘积上限 127 |
| `0x10` | 旋钮 (Knob) | 1 字节：物理控件逻辑索引 |
| `0x11` | 推杆 (Fader) | 1 字节：物理控件逻辑索引 |
| `0x12` | 按钮 (Button) | 1 字节：物理控件逻辑索引 |

**类型码空间与向前兼容：**
- 叶子节点（0x10–0x3F）：遇到未知叶子节点，消耗后续 1 字节索引（共 2 字节），显示为占位符。
- 容器节点（0x01–0x0F）：遇到未知容器节点，必须终止解析并提示用户更新软件。
- 布局树的解析应严格基于 0x08 中声明的长度字段，不应依赖其他字段的值。

### 6.4 示例：上排 4 旋钮，下排 4 推杆
树字节流 22 字节 (0x16)，完整消息：
```
F0 7D 00 08 00 16  02 02 01 04 10 00 10 01 10 02 10 03 01 04 11 04 11 05 11 06 11 07  <校验和> F7
```

## 7. 虚拟控件与库配置（命令 `0x0B` / `0x0C` / `0x0D` / `0x0E`）

### 7.1 查询
```
F0 7D <ID> 0B <校验和> F7
```

### 7.2 响应
```
F0 7D <ID> 0C <库数量 B> <虚拟控件总数 V> <V个虚拟控件描述> <校验和> F7
```
- **B** (1 字节)：库总数 (1–127)
- **V** (1 字节)：虚拟控件总数，有效范围 1–126，且 V ≤ B × N
- **虚拟控件描述**：每个 4 字节：
  - 字节 1：库编号 (0 ~ B-1)
  - 字节 2：物理控件逻辑索引 (0 ~ N-1)
  - 字节 3：CC 号 (0–127)
  - 字节 4：MIDI 通道 (0–15)

### 7.3 设置虚拟配置（写入）
- 命令格式：`F0 7D <ID> 0D <B> <V> <4V字节数据> <校验和> F7`
- 设备逐字段校验：库编号 < B、物理索引 < N、CC ≤ 127、通道 ≤ 15、(库编号, 物理索引) 组合不得重复。
- 校验通过后写入 EEPROM，回复 `0E 00` 成功，否则 `0E 01` 或 `0E 02`。
- 上位机发送后等待 500ms，最多重试 1 次（间隔 400ms）。
- 写入期间设备宜暂停 ADC 扫描与 MIDI 输出，完成后恢复。

## 8. 物理控件校准写入（命令 `0x0F` / `0x10`）

### 8.1 命令
```
F0 7D <ID> 0F <N> <4N字节校准数据> <校验和> F7
```
- **N** (1 字节)：物理控件数量，必须与 0x04 响应一致。
- **校准数据**：每控件 4 字节：`cal_min_mid, cal_min_lo, cal_max_mid, cal_max_lo`，按物理索引 0～N-1 排列。

**校验规则：**
- 总长度必须为 7 + 4N 字节。
- 每条记录的 `cal_max > cal_min`（比较 14-bit 值），否则回复 NACK。
- 按钮控件也应发送校准数据（保持默认值 0 / 4095 即可），固件应忽略按钮的校准值。

### 8.2 应答
- 成功：`F0 7D <ID> 10 00 <校验和> F7`
- 校验失败：`F0 7D <ID> 10 01 <校验和> F7`

## 9. 物理控件 ADC 原始值查询与校准流程

### 9.1 查询 ADC 原始值
```
F0 7D <ID> 11 <校验和> F7
```
设备收到后应立即对所有物理控件进行一次 ADC 采样，并返回原始值。

**时序说明**：设备宜暂停正常 MIDI 输出，完成全部控件采样后一次性返回。采样期间其他 SysEx 可排队处理。若正在执行 EEPROM 写入，应等待写入完成再响应。上位机建议的最小查询间隔为 **50ms**，避免超出设备处理能力。

### 9.2 响应
```
F0 7D <ID> 12 <N> <2N字节原始值> <校验和> F7
```
- **N** (1 字节)：必须与 0x04 一致。
- **原始值数据块**：每控件 2 字节（mid, lo），14-bit 编码，按物理索引顺序排列。

**示例（4 控件，12-bit ADC）**：
```
F0 7D 00 12 04  00 64  1F 40  00 00  1F 7F  0C F7
```
- 控件0：0x64 = 100
- 控件1：0x1F40 (14-bit编码) = (0x1F << 7) | 0x40 = 4032
- 控件2：0x0000 = 0
- 控件3：0x1F7F (14-bit编码) = (0x1F << 7) | 0x7F = 4095
- 校验和 0x0C 已验证。

### 9.3 校准流程

本协议支持两种校准模式，**上位机辅助校准是主要方式**（适用于所有款型），本地按键校准为辅助方式。

#### 模式一：上位机辅助校准（主要）

1. 上位机提示用户“将所有旋钮/推杆调至最小值”（按钮忽略）。
2. 发送 `0x11` 获取各控件的 ADC 原始值，记录为 `cal_min`。为减少机械抖动，建议连续发送 3–5 次（间隔 50ms），取中位数。
3. 提示“将所有旋钮/推杆调至最大值”。
4. 再次发送 `0x11`，记录为 `cal_max`。
5. 验证每个旋钮/推杆的 `cal_max > cal_min`，否则提示用户重试。
6. 通过 `0x0F` 将全部 N 个控件的校准数据写入设备（按钮保持默认值 min=0, max=4095）。
7. 校准期间设备正常发送 MIDI CC，上位机应提示用户暂停使用 DAW，或由上位机临时过滤 CC 输入。

> **注意**：校准流程仅针对旋钮和推杆。按钮的校准值无意义，建议保持默认。上位机可通过布局树中的节点类型（0x10/0x11 vs 0x12）区分。

#### 模式二：本地按键校准（辅助，需设备支持）

设备可设计本地校准功能（如组合键开机），用户按提示操作，设备自动记录并保存。上位机可通过 0x04 查询结果。

## 10. 上位机工作流程

1. 连接后依次发送 `0x03`、`0x07`、`0x0B`。
2. 解析布局和虚拟配置，生成 UI。
3. **校准**：显示当前校准状态，提供“开始校准”按钮，进入模式一校准向导。
4. 用户修改虚拟配置后通过 `0x0D` 写入。
5. 断连重连后重新执行 1–3。

**数据验证**（上位机强制执行）：
- 虚拟控件字段范围和唯一性。
- 布局树叶子索引完整性：旋钮/推杆在每个库中必须存在，按钮可选。
- V ≤ B×N 且 V ≤ 126。

## 11. 下位机实现参考（C 语言）

完整 SysEx 处理框架（包含 0x11 查询）：

```c
#include <stdint.h>

#define MY_DEVICE_ID 0x00
#define N 6          // 物理控件数（4推子+2按钮）
#define B 4
#define V 16

typedef struct {
    uint8_t mux, channel;
    uint8_t cal_min_mid, cal_min_lo;
    uint8_t cal_max_mid, cal_max_lo;
} phy_desc_t;
phy_desc_t phy_desc[N];

typedef struct {
    uint8_t bank, phys_idx, cc, channel;
} virt_control_t;
virt_control_t virt_config[V];
uint8_t current_bank = 0;

// 校验和
uint8_t midi_checksum(const uint8_t *data, int len) {
    int sum = 0;
    for (int i = 0; i < len; i++) sum += data[i];
    return (128 - (sum & 0x7F)) & 0x7F;
}

// 发送缓冲区（能容纳最大 0x04 响应）
static uint8_t tx_buf[770];
static int tx_len;
void tx_begin() { tx_len = 0; }
void tx_byte(uint8_t b) { tx_buf[tx_len++] = b; }
void tx_end() {
    uint8_t ck = midi_checksum(&tx_buf[1], tx_len - 1);
    tx_byte(ck); tx_byte(0xF7);
    uart_send(tx_buf, tx_len);
}

// 解码 14-bit 校准值
uint16_t get_cal_min(uint8_t idx) {
    return ((uint16_t)phy_desc[idx].cal_min_mid << 7) | phy_desc[idx].cal_min_lo;
}
uint16_t get_cal_max(uint8_t idx) {
    return ((uint16_t)phy_desc[idx].cal_max_mid << 7) | phy_desc[idx].cal_max_lo;
}

// ADC 到 MIDI 映射（旋钮/推杆专用）
uint8_t map_adc_to_midi(uint8_t phys_idx, uint16_t adc_raw) {
    uint16_t cal_min = get_cal_min(phys_idx);
    uint16_t cal_max = get_cal_max(phys_idx);
    // 未校准或无效：使用原始缩放（假设 12-bit ADC）
    if (cal_min == 0 && cal_max == 4095) return adc_raw >> 5;   // 0–4095 -> 0–127
    if (cal_max <= cal_min) return 0;
    if (adc_raw <= cal_min) return 0;
    if (adc_raw >= cal_max) return 127;
    return (uint8_t)(((uint32_t)(adc_raw - cal_min) * 127) / (cal_max - cal_min));
}

void handle_sysex(uint8_t *msg, uint16_t len) {
    if (len < 6) return;
    if (msg[0] != 0xF0 || msg[1] != 0x7D || msg[2] != MY_DEVICE_ID) return;
    uint8_t expected = midi_checksum(&msg[1], len - 3);
    if (expected != msg[len-2]) {
        if (msg[3] == 0x0D || msg[3] == 0x0F) send_nack(msg[3] + 1);
        return;
    }

    switch (msg[3]) {
        case 0x03: send_device_info(); break;
        case 0x07: send_layout(); break;
        case 0x0B: send_virt_config(); break;

        case 0x0D:
            if (len != (8 + 4*V) || msg[4] != B || msg[5] != V) { send_nack(0x0E); break; }
            // 字段校验 + 唯一性检查
            // 写入 EEPROM
            send_ack(0x0E, 0x00);
            break;

        case 0x0F:
            if (len != (7 + 4*N) || msg[4] != N) { send_nack(0x10); break; }
            for (int i = 0; i < N; i++) {
                uint16_t cal_min = (msg[5 + i*4] << 7) | msg[6 + i*4];
                uint16_t cal_max = (msg[7 + i*4] << 7) | msg[8 + i*4];
                if (cal_max <= cal_min) { send_nack(0x10); return; }
            }
            // 写入校准 EEPROM
            send_ack(0x10, 0x00);
            break;

        case 0x11: // 查询 ADC 原始值
            tx_begin();
            tx_byte(0xF0); tx_byte(0x7D); tx_byte(MY_DEVICE_ID); tx_byte(0x12);
            tx_byte(N);
            for (int i = 0; i < N; i++) {
                uint16_t raw = read_adc_raw(i);
                tx_byte((raw >> 7) & 0x7F);
                tx_byte(raw & 0x7F);
            }
            tx_end();
            break;
    }
}

void send_ack(uint8_t cmd, uint8_t status) {
    tx_begin();
    tx_byte(0xF0); tx_byte(0x7D); tx_byte(MY_DEVICE_ID);
    tx_byte(cmd); tx_byte(status);
    tx_end();
}
void send_nack(uint8_t cmd) { send_ack(cmd, 0x01); }
```

> **ADC 原始值读取**：`read_adc_raw()` 返回与校准值相同位宽的原始 ADC 读数（不缩放）。对于 12-bit ADC，返回 0–4095；对于 14-bit ADC，返回 0–16383。校准值也使用此原始尺度，故默认 max = 4095 对应 12-bit 满量程。

## 12. 上位机解析参考（JavaScript）

新增辅助函数与完整示例：

```javascript
// 计算校验和
function midiChecksum(data) {
    let sum = 0;
    for (const b of data) sum += b;
    return (128 - (sum & 0x7F)) & 0x7F;
}

// 读取 ADC 原始值
async function readRawADC(deviceId, midiOutput, midiInput) {
    const request = [0xF0, 0x7D, deviceId, 0x11];
    const ck = midiChecksum(request.slice(1));
    midiOutput.send([...request, ck, 0xF7]);

    // 等待 0x12 响应（500ms 超时）
    const response = await waitForSysex(midiInput, 0x12, deviceId, 500);
    const N = response[4];
    const rawValues = [];
    for (let i = 0; i < N; i++) {
        const off = 5 + i * 2;
        rawValues.push((response[off] << 7) | response[off + 1]);
    }
    return rawValues;
}

// waitForSysex 需实现 MIDI 事件监听并过滤
```

同前。

### 13.2 当前库编号不持久化
同前。

### 13.3 虚拟控件查找采用线性遍历
同前。

### 13.4 校验和算法沿用 Roland 式求和
同前。

### 13.5 物理映射只读
同前。

### 13.6 校准数据采用 14-bit 编码
已覆盖 12/14-bit ADC，未来 16-bit ADC 可通过新命令扩展。

### 13.7 本地按键校准为辅助方式
多数款型无专用按钮，上位机辅助校准是主要方式。设备可保留本地按键校准作为辅助。

### 13.8 按钮的校准数据处理
按钮采用阈值判断而非线性映射，校准值无意义。0x0F 写入时保持默认值。

### 13.9 校准期间的静音模式（未来扩展）
当前校准期间设备仍发送 MIDI CC，可能干扰 DAW。未来可引入“静音模式”命令（如 0x13），临时暂停 MIDI 输出。
